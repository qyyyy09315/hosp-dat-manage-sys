# 医院数据管理系统 (Hosp-Dat-Manage-Sys)
<div align="center">
Java Swing 桌面端医院信息管理应用 | 基于RBAC权限模型 | 密码哈希加密 | 全链路审计日志
</div>

---

## 摘要
本项目是一款面向中小医疗机构的轻量级桌面信息管理系统，采用经典客户端-数据库两层架构，实现了病人信息管理、医生排班与绩效统计、药品库存维护、分级权限控制、操作审计追踪等核心功能。系统基于2024-2025学年Java实验课期末大作业扩展开发，融合了《大数据安全》课程的核心防护理念，在可用性、安全性、可扩展性方面做了针对性优化，可作为Java桌面应用开发、医疗信息系统安全研究的教学参考实例。

---

## 项目背景与演进
本项目的开发过程分为两个阶段：
1. **基础功能开发阶段 (2024.10-2025.01)**：作为《Java程序设计》课程期末大作业，完成了核心业务功能（病人/医生/药品管理）、Swing UI界面、JDBC数据库交互、基础权限校验模块，通过课程验收。
2. **安全增强阶段 (2025.02-2025.06)**：融合《大数据安全》课程知识点，对系统安全模块进行系统性升级，引入BCrypt密码哈希存储、RBAC五表权限模型、全链路操作审计、SQL注入防护等特性，达到教学级安全标准。

## 安全特性
本系统针对医疗数据的敏感性，实现了多层次安全防护：
1. **密码安全**：采用BCrypt算法对用户密码进行加盐哈希存储，迭代轮数设为10，抗暴力破解与彩虹表攻击
2. **分级权限控制**：基于RBAC（角色-based访问控制）模型，实现<用户-角色-权限>三级映射，细粒度控制菜单与操作权限
3. **操作审计**：所有用户操作（登录/查询/增删改）均记录审计日志，包含操作人、时间、IP、内容、状态，支持事后追溯
4. **输入校验**：前端实现非空校验、格式校验、特殊字符过滤，防止SQL注入与XSS攻击
5. **数据库最小权限**：建议使用专用业务账号访问数据库，仅授予SELECT/INSERT/UPDATE/DELETE权限，禁止DROP/ALTER等高危操作

## 部署与运行指南
### 环境要求
- JDK 17 及以上
- MySQL 8.0 及以上
- Maven 3.8 及以上

### 部署步骤
1. **数据库初始化**：
   ```sql
   -- 按顺序执行SQL脚本
   source sql/init_security_tables.sql;
   source sql/init_business_tables_and_data.sql;
   source sql/init_test_users.sql;
   ```
2. **配置数据库连接**：修改`src/main/java/com/sxt/common/DBUtils.java`中的数据库连接信息
3. **项目打包**：
   ```bash
   mvn clean package -DskipTests
   ```
4. **运行应用**：
   ```bash
   java -jar target/hospital-management-system-1.0-SNAPSHOT-jar-with-dependencies.jar
   ```
